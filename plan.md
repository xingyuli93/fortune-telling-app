# 后端服务开发计划

## 目标

构建一个强大、可扩展的后端服务，作为“算命引擎”，为前端应用（微信小程序、网页）提供基于用户输入（姓名、生日、MBTI）的、多维度的、富有深度和文采的算命结果。

---

## 阶段一：环境设置与基础架构 (地基)

*   [ ] **任务1.1：技术选型确认**
    *   编程语言：Python (推荐，便于处理复杂逻辑和数据分析)
    *   Web 框架：FastAPI (推荐，高性能，异步支持，自动生成API文档)
    *   数据库：PostgreSQL (推荐，功能强大，支持复杂查询和JSON数据类型)

*   [ ] **任务1.2：开发环境搭建**
    *   安装 Python 和 Poetry (或 Pip) 进行包管理。
    *   安装并配置 PostgreSQL 数据库。
    *   初始化 FastAPI 项目结构。

*   [ ] **任务1.3：创建项目骨架**
    *   建立清晰的目录结构，例如：`app/`, `core/`, `models/`, `tests/`。
    *   配置基础的日志记录和错误处理机制。

---

## 阶段二：核心算法与数据模型 (灵魂)

*   [ ] **任务2.1：定义数据模型 (Schema)**
    *   设计 `fortunes` (签文) 表结构，包含签文ID、签文本身、以及多维度解读 (summary, study, career, love, health, wealth, social 等)。
    *   设计 `knowledge_base` (知识库) 表结构，用于存储五行、天干地支、MBTI性格分析等基础数据。

*   [ ] **任务2.2：实现核心计算逻辑**
    *   开发一个 `bazi.py` 模块，用于根据公历生日计算生辰八字、天干地支、五行属性。
    *   开发一个 `mbti.py` 模块，内置MBTI性格特征分析文本。
    *   (可选) 开发一个 `naming.py` 模块，用于分析姓名笔画和三才五格。

*   [ ] **任务2.3：开发“算命引擎”**
    *   创建一个 `engine.py` 模块，这是核心中的核心。
    *   它将调用上述模块，整合用户的八字、五行、MBTI等信息，形成一个综合的用户画像 (Profile)。
    *   根据这个画像，设计一套加权匹配算法，从数据库中查询出最符合的几条签文作为候选。

---

## 阶段三：API 接口开发 (桥梁)

*   [ ] **任务3.1：设计并实现 API Endpoint**
    *   创建一个 `POST /api/v1/divine` 接口。
    *   请求体 (Request Body) 接收 `{ "name": "张三", "birthdate": "1990-01-01", "mbti": "INFJ" }`。
    *   响应体 (Response Body) 返回结构化的JSON，包含签文和所有维度的分析。

*   [ ] **任务3.2：编写 API 文档**
    *   利用 FastAPI 的特性，自动生成并完善 OpenAPI (Swagger) 文档，方便前端调用和未来维护。

*   [ ] **任务3.3：前端对接**
    *   修改小程序和网页的 `getFortune` 函数，将本地模拟逻辑替换为调用此API的真实网络请求。

---

## 阶段四：数据填充与部署 (血肉与发布)

*   [ ] **任务4.1：扩充数据库**
    *   编写脚本，将收集到的大量签文、解读、知识库数据批量导入到数据库中 (目标：1000+条)。

*   [ ] **任务4.2：容器化**
    *   创建 `Dockerfile`，将Python应用打包成一个独立的Docker镜像。

*   [ ] **任务4.3：部署**
    *   选择一个云平台 (如 腾讯云、阿里云、Vercel 等)。
    *   将 Docker 镜像部署到云服务器或 Serverless 平台，并配置 API 网关。

*   [ ] **任务4.4：压力测试与优化**
    *   对部署好的API进行压力测试，确保其在高并发下的稳定性和响应速度。

---
